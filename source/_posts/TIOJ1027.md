## what is 牛頓法 （Newton's Method）

### 概論

這個方法是給定一個**凹凸性固定**的函數，找根的方法。它是一個從一個隨便給定的猜測（有點像二分搜一開始的`m`）開始，然後逐步將這個猜測變得越來越好的過程。

### 越猜越準的過程

註：這邊的東西我也不大會證明，只要自己感覺是對的就好，拍謝 >< 

給定一個函數$f(x)$，我們想要找到一個$x$使得$f(x) = 0$。雖然$x$很難找，但是我們可以找一個一開始的猜測$x_0$，然後從$x_0$開始。我們可以計算$f$在$x_0$的微分$m = f'(x_0)$，並且構造一條斜率$m$且通過$(x_0, f(x_0))$的線：
$$L: (y - f(x_0)) = m(x - x_0)$$
而$L$就是$f$的線性近似。而雖然$f$很難找到根，但是$L$的根（$x$-截距）就很好找了，在$x_1 = x_0 - \frac{f(x_0)}{m}$的地方。如果一直重複這個算法，用相同的算法計算$x_2$、$x_3$、$\dots$，那你就可以近似真正的根$a$至任何的精準度了$^1$。也就是說，如果我們現在有一個可微的函數$f$和目前的猜測$x_t$，我們可以透過做線性近似然後找根來找到下一個猜測$x_{t + 1}$：
$$x_{t + 1} = x_t - \frac{f(x_t)}{f'(x_t)}$$
我覺得下面的動畫做得很好，在沒有嚴謹證明時候就請用這張圖來說服自己吧！

### 比二分搜還厲害？

如果滿足一些條件的話（請看下面證明），那他的收斂理論上會很快。具體來說，令$\varepsilon_k = |a  - x_k|$為第$k$步時的誤差。則
$$\varepsilon_k = O(\varepsilon_{k - 1})$$，也就是說理論上每一步，正確的位數就會翻倍！不過，實際上也沒有那麼好，因為一開始的收斂就會比較慢。

[維基百科較細證明](https://en.wikipedia.org/wiki/Newton%27s_method#Proof_of_quadratic_convergence_for_Newton's_iterative_method)

實際上來說，我測試是差不多的，二分搜差不多要跑$52 \log_{2}10 \approx 172$次而這個牛頓法差不多需要$183$次，就其實沒有少很多XD

### 在這一題上的應用

輸入$n$的話，我們的目標函數就是$f(x) = x^2 - n$，可以計算$f'(x) = 2x$。一開始隨便選一個$x_0 = n$就好了。也就是說，
$$x_k = \begin{cases} 
\begin{array}{c c}
    n, & k= 0\\\\
    \frac{x^2_{k - 1} + n}{2x_{k - 1}}, & k > 0\\\\
\end{array}
\end{cases}$$
這樣去迭代就好了。至於迭代幾次，傳了調一下就知道了XD

## AC Code 
是不是很短呢！
```haskell
import Data.Ratio

main = do
   line <- getLine
   let 
        num = read (line) :: Integer
        sq = (getSqrt num)
        str = show sq
        (nume, floa) = splitAt ((length str) - 50) str
   putStrLn (nume ++ "." ++ floa)

run n 0 = n
run n t = 
    (n + y * y) `div` (2 * y) 
    where y = run n (t - 1)

getSqrt x 
    | (y `mod` 10) >= 5 = 
        (y) `div` 10 + 1
    | otherwise = y `div` 10
    where y = run (x * 10^102) 183

```
